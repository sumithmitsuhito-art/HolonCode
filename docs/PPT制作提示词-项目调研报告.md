## 第 7 页：技能系统详解

标题：**技能系统 — AI 的"可插拔能力模块"**

内容要点：

**什么是技能？**
- 技能是一个 Markdown 文件（SKILL.md），里面用自然语言描述了"当激活这个技能时，AI 应该如何表现"
- 存放在 `skills/<技能名>/SKILL.md`
- AI 自己可以创建、修改、激活、关闭技能——实现自我进化

**技能的工作机制：**
- 技能目录（只显示名称和描述）拼入系统提示词
- AI 判断需要某个技能时，调用 activate_skill 激活
- 激活后，技能的完整行为指令注入系统提示词
- 最多同时激活 3 个技能，超出时自动关闭最早的
- 不再需要时调用 deactivate_skill 关闭

**内置技能示例：**
- code-reviewer — 傲娇编程导师，边教边吐槽
- translator — 中/英/日三语翻译
- storyteller — 交互式故事创作
- confidant — 深度倾听与共情

设计要求：左侧放"技能是什么"的解释，右侧放工作流程图（目录→激活→注入→关闭），下方放四个技能卡片。

## 第 9 页：选题意义

标题：**选题意义**

内容要点（每点配一个简约图标或几何图形）：

**1. 探索 LLM Agent 的工程化实践**
Function Calling 是当前 AI 应用开发的核心范式。本项目完整实现了"工具定义 → AI 决策 → 工具执行 → 结果反馈"的闭环，是学习 AI Agent 开发的优秀实践案例。

**2. 让 AI 拥有"长期记忆"与"自我进化"能力**
用户画像系统让 AI 越来越懂你，技能系统让 AI 能自我扩展能力边界——这两个设计让对话机器人从"一次性工具"变成"会成长的伙伴"。

**3. 轻量级、零门槛部署**
只用 Python 标准库 + 2 个第三方库，不依赖数据库、Web 框架、Docker。任何装了 Python 的电脑都能跑。

**4. 可扩展的架构设计**
角色设定、行为准则、工具列表全部通过 JSON/Markdown 文件配置，不需要改代码就能深度定制。
